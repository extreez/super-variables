# Super Variables — Figma Plugin: План реализации

## Обзор
Плагин для Figma по управлению переменными с двусторонней синхронизацией (экспорт/импорт), табличным UI похожим на нативные Figma Variables, и продвинутыми возможностями для архитекторов дизайн-систем.

## 🏗️ Архитектура

```
super-variables/
├── manifest.json          # Figma plugin manifest
├── package.json           # Dependencies
├── tsconfig.json          # TypeScript config
├── src/
│   ├── code.ts            # Plugin sandbox (Figma API)
│   ├── ui.html            # UI entry point
│   ├── ui/                # UI layer (vanilla JS + HTML + CSS)
│   │   ├── app.ts         # Main UI controller
│   │   ├── styles/        # CSS файлы
│   │   ├── components/    # UI компоненты
│   │   └── utils/         # UI утилиты
│   ├── core/              # Бизнес-логика (shared)
│   │   ├── types.ts       # Типы данных
│   │   ├── parser/        # Парсеры (CSS, JSON, SCSS...)
│   │   ├── generator/     # Генераторы (CSS, JSON, SCSS...)
│   │   ├── mapper/        # Маппинг имён
│   │   └── diff/          # Сравнение и merge
│   └── bridge/            # Коммуникация code ↔ UI
│       └── messages.ts    # Типы сообщений
└── dist/                  # Скомпилированный output
```

## 🔧 Технологии
- **TypeScript** — основной язык
- **Vanilla CSS + HTML** — UI (без фреймворков, для минимального bundle size в Figma)
- **esbuild** — быстрая сборка
- **Figma Plugin API** — работа с переменными

## Правила разделителей (Naming Convention)
По умолчанию:
- `--` (двойной дефис) = разделитель групп/подгрупп → `/` в Figma
- `-` (одинарный дефис) = часть имени (как пробел)
- Пример: `--color--brand--primary-500` → `color/brand/primary-500`
- Кастомизируемо через настройки
