# Гайд по тестированию функций Импорта и Экспорта (Super Variables)

Этот документ содержит пошаговый сценарий для проверки корректности работы плагина. Для тестов используйте пустой файл Figma или отдельную страницу.

---

## 1. Подготовка эталонных данных (Setup)

Перед началом создайте в Figma вручную следующую структуру для проверки всех типов данных:

1. **Коллекция: "Primitive"**
   - Группа `Color`: 
     - `blue-500` (Color: `#0000FF`)
   - Группа `Spacing`: 
     - `base` (Float: `16`)
2. **Коллекция: "Theme"** (2 режима: `Light` и `Dark`)
   - `surface-primary` -> Алиас на `Primitive/Color/blue-500`
   - `is-dark` (Boolean: `false` в Light, `true` в Dark)
   - `text-main` (String: "Hello")

---

## 2. Сценарий 1: Базовый Экспорт и Проверка Форматов

**Цель:** Убедиться, что плагин правильно читает данные и формирует файл.

1. Откройте плагин -> Нажмите **Export**.
2. Выберите формат **JSON (DTCG)**.
3. **Проверьте:** в превью должны быть видны пути `Primitive/Color/blue-500` и метаданные (scopes, если они заданы).
4. Переключитесь на **CSS**.
5. **Проверьте:** алиасы должны экспортироваться как `var(--primitive-color-blue-500)`.
6. Сохраните JSON файл (или скопируйте содержимое).

---

## 3. Сценарий 2: "Чистый" Импорт (Создание с нуля)

**Цель:** Проверить создание структуры в пустом файле.

1. Удалите все переменные из файла Figma (или создайте новый файл).
2. Откройте плагин -> **Import**.
3. Вставьте JSON, полученный в Сценарии 1.
4. В настройках импорта:
   - `Strategy`: **Auto** (плагин сам создаст коллекции по именам в JSON).
   - `Use Custom IDs`: **Off**.
5. Нажмите **Import**.
6. **Результат:** Плагин должен воссоздать коллекции "Primitive" и "Theme", все группы, режимы (Light/Dark) и правильно связать алиасы.

---

## 4. Сценарий 3: Переименование и Custom ID (Синхронизация)

**Цель:** Проверить, что плагин понимает, что это та же переменная, даже если имя изменилось.

1. В Figma переименуйте `Primitive/Color/blue-500` в `Primitive/Color/brand-blue`.
2. Сделайте Экспорт в JSON с включенной опцией **Include Custom IDs**.
3. В JSON-файле (в текстовом редакторе) вручную измените имя: `"name": "Primitive/Color/super-blue"`.
4. Сделайте Импорт этого JSON, включив **Use Custom IDs** в настройках.
5. **Результат:** Плагин НЕ должен создать новую переменную. Он должен переименовать существующую `brand-blue` обратно в `super-blue`, так как их `customId` совпали.

---

## 5. Сценарий 4: Работа с Конфликтами (Manual Mapping)

**Цель:** Проверить принудительную загрузку данных в конкретную коллекцию.

1. Возьмите JSON с токенами из коллекции "Primitive".
2. В окне Импорта выберите:
   - `Strategy`: **Manual**.
   - `Target Collection`: Выберите "Theme" (или создайте новую).
   - `Target Mode`: Выберите конкретный режим.
3. **Результат:** Все импортируемые токены должны попасть в выбранную коллекцию/режим, игнорируя пути, прописанные в JSON.

---

## 6. Сценарий 5: Scopes и Специфичные настройки

**Цель:** Проверка ограничений использования переменных.

1. В Figma у переменной `Primitive/Spacing/base` снимите все галочки в "Edit Variable" -> "Scopes" (например, оставьте только "Gap").
2. Экспортируйте в JSON.
3. Удалите переменную в Figma.
4. Импортируйте JSON обратно.
5. **Результат:** Проверьте настройки Scopes у восстановленной переменной. Они должны в точности соответствовать исходным.

---

## 7. Сценарий 6: Конфиг изменений (Changes Log)

**Цель:** Проверка массового удаления и переименования через специальный блок `changes` в импорте.

1. Подготовьте JSON с дополнительным полем `changes`:
```json
{
  "tokens": [...],
  "changes": {
    "deletions": ["Primitive/Spacing"],
    "renames": {
      "Theme/text-main": "Theme/typography/body-text"
    }
  }
}
```
2. Выполните Импорт.
3. **Результат:** 
   - Вся группа `Primitive/Spacing` должна быть удалена из Figma.
   - Переменная `text-main` должна переместиться в новую группу `typography`.

---

## 8. Чек-лист успешной проверки:
- [ ] Алиасы не "ломаются" и ведут на правильные токены.
- [ ] Цвета передаются точно (HEX/RGBA).
- [ ] Числа (Float) корректно обрабатывают `px` и `rem` (если baseFontSize = 16).
- [ ] Режимы (Modes) создаются автоматически, если их нет.
- [ ] Окно лога импорта показывает: сколько добавлено, обновлено и ошибок.
